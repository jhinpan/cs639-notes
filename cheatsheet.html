CS 639 Foundation Models — Midterm Cheat Sheet

1. ML Fundamentals

  • Supervised learning: input $\mathcal{X}$, labels $\mathcal{Y}$, hypothesis $h \in \mathcal{H}$, minimize empirical risk $\frac{1}{n}\sum_i \ell(h(x_i), y_i)$
  • Bias-Variance: $\text{Error} = \text{Bias}^2 + \text{Variance} + \text{Noise}$. ↑ complexity → ↓ bias, ↑ variance
  • Regularization: L1 (sparsity), L2/weight decay (small weights), Dropout (randomly zero activations)
  • Logistic regression: $\sigma(z) = \frac{1}{1+e^{-z}}$, cross-entropy loss: $-[y\log\hat{y} + (1-y)\log(1-\hat{y})]$

2. Deep Learning

MLP / Perceptron

  • $\mathbf{h} = \sigma(W^{(1)}\mathbf{x} + \mathbf{b}^{(1)})$, output: $\hat{y} = W^{(2)}\mathbf{h} + \mathbf{b}^{(2)}$
  • Activations: ReLU $\max(0,x)$, GELU $x\Phi(x)$, Tanh, Sigmoid
  • XOR: not linearly separable → need hidden layers

CNN

  • Output size: $(n_h - k_h + 2p + 1) \times (n_w - k_w + 2p + 1)$ with stride 1, padding $p$
  • Properties: local connectivity, weight sharing, translation equivariance

RNN / LSTM

  • RNN: $s^{(t)} = \tanh(Ws^{(t-1)} + Ux^{(t)} + b)$
  • Problem: vanishing/exploding gradients for long sequences
  • LSTM: forget gate $f_t = \sigma(W_f[h_{t-1},x_t]+b_f)$, input gate $i_t$, output gate $o_t$, cell state $c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$

Training

  • Backprop: chain rule through comp. graph. $\frac{\partial L}{\partial w} = \frac{\partial L}{\partial z}\frac{\partial z}{\partial w}$
  • SGD: $w \leftarrow w - \eta \nabla_w L$; Adam: adaptive LR + momentum
  • Residual connection: $y = F(x) + x$ (gradient highway, enables deep nets)
  • BatchNorm: normalize across batch dim; LayerNorm: normalize across feature dim (used in Transformers)

3. Self-Supervised Learning

Core idea: Create supervision from unlabeled data via pretext tasks. No human labels needed.

Masking (BERT-style)

  • Mask ~15% tokens: 80% → [MASK], 10% → random, 10% → unchanged
  • Loss: $\mathcal{L}_{\text{MLM}} = -\sum_{i \in \mathcal{M}} \log P(x_i \mid x_{\setminus\mathcal{M}})$
  • Bidirectional context → good for understanding tasks

Autoregressive (GPT-style)

  • Next-token prediction: $P(w_1,...,w_n) = \prod_i P(w_i \mid w_1,...,w_{i-1})$
  • Unidirectional (causal) → good for generation

Contrastive Learning

  • Pull positive pairs close, push negatives apart
  • InfoNCE: $\mathcal{L} = -\log\frac{\exp(\text{sim}(z_i,z_j)/\tau)}{\sum_{k\neq i}\exp(\text{sim}(z_i,z_k)/\tau)}$
  • SimCLR: augment → encode → project → contrastive loss. Needs large batch.
  • MoCo: momentum encoder + queue of negatives

Student-Teacher

  • BYOL, DINO: teacher = EMA of student: $\theta_T \leftarrow \alpha\theta_T + (1-\alpha)\theta_S$
  • No negative samples needed; avoids collapse via EMA + centering

4. Attention & Transformers

Core formula: $\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$
  • QKV: $Q = XW_Q$, $K = XW_K$, $V = XW_V$; $W_Q, W_K \in \mathbb{R}^{d \times d_k}$
  • Why scale by $\sqrt{d_k}$? Dot products grow with $d_k$ → softmax saturates → tiny gradients
  • Multi-head: $\text{MHA}(Q,K,V) = \text{Concat}(\text{head}_1,...,\text{head}_h)W^O$, each head dim = $d_k = d_{\text{model}}/h$
  • Self-attention: Q, K, V all from same sequence → permutation-invariant!
  • Cross-attention: Q from decoder, K/V from encoder

Positional Encoding

  • Needed because self-attention is permutation-invariant
  • Sinusoidal (original): $PE_{(pos,2i)} = \sin(pos/10000^{2i/d})$, $PE_{(pos,2i+1)} = \cos(pos/10000^{2i/d})$
  • Learned (BERT): trainable embedding per position
  • RoPE: rotate Q,K by position angle → dot product encodes relative position $m-n$

Transformer Block

$x' = \text{LayerNorm}(x + \text{MHA}(x))$
$\text{output} = \text{LayerNorm}(x' + \text{FFN}(x'))$
$\text{FFN}(x) = W_2 \cdot \text{GELU}(W_1 x + b_1) + b_2$, typically $d_{ff} = 4d_{\text{model}}$
$\text{LayerNorm}(x) = \gamma \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta$ (per-sample, across features)

Causal mask: lower-triangular matrix. Row $i$: non-negative, sums to 1, upper triangle = 0. Token $i$ only sees tokens $\leq i$.

6. Attention Variants

  • Standard: $O(n^2)$ time and memory for sequence length $n$
  • Sparse attention: local window + global tokens (Longformer, BigBird). Global tokens = info hub for long-range.
  • Linear attention: kernel trick $\phi(Q)\cdot[\phi(K)^\top V]$ → $O(n)$. Compute KV first, avoid $n\times n$ matrix.
  • GQA: group query heads share K/V heads → smaller KV cache for inference
  • Flash Attention: IO-aware tiling. Exact attention (not approximate). Tiles Q,K,V blocks in SRAM; avoids materializing full $n\times n$ in HBM. Online softmax trick.
  • State Space Models (S4/Mamba): $h'(t)=Ah(t)+Bx(t)$, $y(t)=Ch(t)$. Linear recurrence → efficient for long sequences.

7. Multimodal Models

Vision Transformer (ViT)

  • Image → fixed-size patches (e.g. 16×16) → linear projection → patch embeddings
  • Prepend [CLS] token + add position embeddings → standard Transformer encoder
  • "An image is worth 16×16 words"

CLIP (Contrastive Language-Image Pre-training)

  • Dual encoder: image encoder + text encoder → shared embedding space
  • Contrastive loss on $N\times N$ similarity matrix: diagonal = positives, off-diagonal = negatives
  • Zero-shot classification: encode text prompts ("a photo of a {class}") + image → cosine similarity
  • Trained on 400M image-text pairs from the web

  • Perceiver Resampler: compress variable visual tokens → fixed number
  • Gated cross-attention inserted between frozen LM layers
  • Only trains new cross-attention params + resampler (LM stays frozen)
  • Enables few-shot multimodal ICL with interleaved image-text

8. Prompting & In-Context Learning

Prompting Basics

TypeDescription
Zero-shotTask instruction only, no examples
One-shot1 input-output example + test input
Few-shotK examples (K=2-5 typical) + test input
  • Prompt sensitivity: format, example choice, example ORDER all significantly affect performance
  • ICL: model "learns" from in-prompt examples without parameter updates. Emergent ability at scale.

Advanced Prompting

Chain-of-Thought (CoT): Include step-by-step reasoning in examples. Model generates intermediate reasoning steps before answer. Dramatically improves math/logic tasks. More "thinking time" = better reasoning.
  • Zero-shot CoT: Just add "Let's think step by step" to the prompt
  • Self-consistency: sample K reasoning paths → majority vote on final answer

Soft Prompts / Prompt Tuning

  • Replace discrete text prompts with continuous learnable vectors in embedding space
  • Only optimize prompt embeddings via gradient descent; model params frozen
  • Prefix-Tuning: prepend trainable prefix vectors to K,V at every layer
  • Advantages: continuous search space; tiny params; task-specific "adapters"

10. Common Exam Traps

  • BERT ≠ decoder: BERT is encoder-only, no causal mask, no cross-attention
  • GPT ≠ encoder: GPT is decoder-only with causal mask
  • Positional encoding ≠ causal mask: PE gives position info; causal mask prevents future tokens
  • Scale factor: divide by $\sqrt{d_k}$ (not $d_k$, not $d_{\text{model}}$)
  • Causal attention matrix: lower triangular + non-negative + rows sum to 1
  • Cross-attn: Q from decoder, K/V from encoder (not the reverse!)
  • CLIP: contrastive (not generative), dual encoder (not single stream)
  • Flash Attention: exact (not approximate), IO optimization (not algorithmic change)
  • Linear Attention: kernel replaces softmax → reorder matmul → avoid $n^2$
  • ICL: no parameter updates during inference; learning happens via forward pass
  • Soft prompts: continuous vectors, optimized by gradient descent (not text search)
  • Foundation Model: pretrained at scale + adaptable to downstream tasks
